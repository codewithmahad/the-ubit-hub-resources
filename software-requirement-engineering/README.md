# Software Requirement Engineering

**SE453 — Study Archive**

</div>

---

> **Software Requirement Engineering (SRE)** is the discipline of systematically eliciting, analyzing, specifying, validating, and managing software requirements before a single line of code is written. This folder contains every resource used during the SE453 course — organized so that any student, from any degree program (CS, SE, AI) and any semester, can pick it up and prepare effectively.

## Topics Covered

Based on the SE453 course outline, the following areas are covered across the study materials in this folder:

| Area | Topics |
|------|--------|
| **Foundations** | Intro to RE, requirement types (functional/non-functional), RE processes |
| **Elicitation** | Stakeholder analysis, elicitation techniques, interviews, prototyping |
| **Analysis** | Trade-off analysis, quality requirements, use cases, conflict resolution |
| **Specification** | Writing SRS documents, documentation standards, natural language vs. formal |
| **Validation** | Reviews, prototyping for validation, requirement walkthroughs |
| **Management** | Traceability, prioritization, change management, risk & impact analysis |
| **Advanced** | RE in agile methods, RE in systems engineering, architecture interaction |
